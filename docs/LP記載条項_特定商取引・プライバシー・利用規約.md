# 個人開発者 LP（サブスクあり）で必要な記載条項の洗い出し

本プロジェクトは **ランディングページ（LP）＋ 有料サブスクリプション（Stripe）＋ Google OAuth（Supabase Auth）** を利用しているため、以下の条項・ページの整備が求められます。

---

## 1. 必須で用意すべきページ・文書一覧

| 種類 | 必要性 | 主な根拠・用途 |
|------|--------|----------------|
| **特定商取引法に基づく表記** | 必須 | 有料サブスク＝「役務の提供を受ける契約」であり、電子消費者契約・通信販売に該当するため義務 |
| **プライバシーポリシー** | 必須 | 個人情報取扱・Google OAuth 審査要件・Stripe/Supabase 利用に伴うデータ処理の明示 |
| **利用規約** | 強く推奨 | 利用条件・禁止事項・サブスク解約・免責・知的財産・準拠法の明文化 |
| **キャンセル・返金ポリシー** | 推奨 | 特定商取引の「返品・解約」欄と連動。サブスク解約手順の明示 |
| **Cookie・追跡の説明** | 状況により | 分析・広告 Cookie を使う場合はプライバシーポリシー内または別途明示 |

---

## 2. 特定商取引法に基づく表記

有料サブスクは「**役務の提供を受ける契約**」に該当し、**電子消費者契約**および**通信販売**として、特定商取引法に基づく表記が義務付けられます。

### 2.1 記載すべき項目（抜粋）

- **販売業者・事業者**
  - 事業者名（個人の場合は氏名または屋号）
  - 代表者名（個人事業主の場合は氏名）
  - 所在地・連絡先（メールアドレス等）
- **販売価格**
  - 月額・年額の税込表示、支払方法（クレジットカード等）
- **代金の支払時期・方法**
  - 例：毎月〇日 / 毎年〇月〇日、Stripe 経由のクレジットカード決済
- **役務の提供時期**
  - 例：決済完了後ただちに Pro 機能を利用可能に
- **返品・解約**
  - サブスクの解約方法（Stripe カスタマーポータル等）、解約後の日割り返金の有無、最低利用期間の有無
- **動作環境・不具合時の対応**
  - 動作環境の記載、問い合わせ窓口
- **その他**
  - 特定商取引法で定める「役務の提供を受ける契約」に係る表記（電子消費者契約・クーリングオフ等の適用の有無）

### 2.2 表記の置き場所

- LP のフッターなど、**購入前に必ず見られる場所**に「特定商取引に基づく表記」へのリンクを設置
- チェックアウト（Stripe Checkout）前に「利用規約・特定商取引に同意する」を表示する運用も有効

## 3. プライバシーポリシー

個人情報保護法・Google OAuth 審査・Stripe/Supabase 利用の観点から必須です。

### 3.1 記載すべき内容

- **取得する情報**
  - 本サービスで取得するデータの例：
    - **認証**: Google アカウントによるログイン（Supabase Auth）→ メールアドレス、表示名、プロフィール画像（Google から提供される範囲）
    - **決済・サブスク**: Stripe 経由の決済情報（カード情報は Stripe が保持、事業者は Stripe Customer ID・サブスク状態等を保持）
  - Supabase に保存するデータ：`user_id`、`stripe_customer_id`、`stripe_subscription_id`、`status`、`plan`、`current_period_end` 等
- **利用目的**
  - 本人認証、サブスク状態の管理、問い合わせ対応、サービス改善
- **第三者提供**
  - Google（認証）、Stripe（決済）、Supabase（ホスティング・DB）への提供があることを明記。各社のプライバシーポリシーへのリンクを推奨
- **Cookie・ローカルストレージ**
  - 使用する場合は目的と種類を記載（Supabase のセッション等）
- **保持期間・削除**
  - アカウント削除時や解約後のデータ保持期間、削除請求への対応方針
- **お問い合わせ**
  - 開示・訂正・削除等の請求窓口（メールアドレス等）
- **改定**
  - ポリシー改定時の告知方法（サイト掲示・メール等）

### 3.2 Google OAuth 審査で必要なこと

- **プライバシーポリシーの URL を公開済み LP 上に用意する**
- 同意画面の「プライバシーポリシー」欄にその URL を設定してから審査申請する（SETUP_GUIDE にも記載あり）

## 4. 利用規約

サービスの利用条件を定め、トラブル防止と免責の根拠にします。

### 4.1 記載を推奨する項目

- **適用・同意**
  - 本規約の適用範囲、利用開始をもって規約に同意したとみなす旨
- **利用資格**
  - 行為能力、禁止されている利用（転売・再配布・逆コンパイル等）
- **アカウント・認証**
  -  Google アカウントによるログイン、アカウントの管理責任
- **有料プラン（サブスク）**
  - 料金・支払方法・更新（自動更新）・解約方法（Stripe カスタマーポータル等）
  - 解約後の Pro 機能の利用終了時期、日割り返金の有無
- **禁止事項**
  - 法令違反、不正利用、権利侵害、リバースエンジニアリング等
- **知的財産**
  - サービス上のコンテンツ・ソフトウェアの著作権・利用許諾の範囲
- **免責**
  - サービスは「現状のまま」提供、動作保証の範囲、間接損害の免責
- **契約解除**
  - 規約違反時の利用停止・解約、こちらの判断によるサービス終了の可能性
- **規約変更**
  - 変更手続き・効力発生（掲示またはメール等）
- **準拠法・管轄**
  - 日本法準拠、裁判管轄（例：事業者所在地の裁判所）

---

## 5. キャンセル・返金ポリシー（サブスク解約）

- **解約方法**
  - アプリ内または Stripe カスタマーポータルから解約できる旨、手順の簡潔な説明
- **解約の効力**
  - 解約後は当期満了まで Pro 利用可能、翌期から Free に戻る旨
- **返金**
  - 日割り返金を行うかどうか（行わない場合は「返金は行いません」と明記）
- **最低利用期間**
  - ある場合は明記（例：初回〇ヶ月は解約不可等）

これらは「特定商取引に基づく表記」の「返品・解約」と内容を揃えておくとよいです。

## 6. LP 上での配置・リンクの目安

- **フッター**
  - プライバシーポリシー | 利用規約 | 特定商取引に基づく表記 | お問い合わせ
- **購入ボタン付近**
  - 「ご購入には利用規約・特定商取引に基づく表記への同意が必要です」等の一文＋リンク
- **Google OAuth 審査**
  - アプリのホームページ URL：LP のトップ
  - プライバシーポリシー URL：上記フッターのプライバシーポリシーページ
